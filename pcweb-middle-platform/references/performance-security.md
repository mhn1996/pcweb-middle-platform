# 性能与安全

用于性能敏感工作、权限行为、鉴权变更和安全评审。

## 加载性能

- 非首屏页面使用路由级懒加载。
- 首屏代码聚焦必要功能。
- 懒加载非关键组件和重型可视化。
- 避免不必要的大型依赖。
- 优先使用可 tree-shaking 的 imports。
- 压缩和优化图片。

## 运行时性能

- 在添加大量 memoization 前，先避免不必要的重新渲染。
- 当存在可测量或明显的渲染成本时，使用 `React.memo`、`useMemo` 和 `useCallback`。
- 当数据量需要时，大型表格/列表使用虚拟滚动。
- 组件卸载时清理定时器、订阅、监听器和未完成请求。
- 派生数据计算较复杂时，将其放在 JSX 外部。

## 请求性能

- 避免页面区块之间重复请求。
- 对重复数据使用共享 hooks 或 service 级规范化。
- 可用时，通过合并接口获取基础数据/枚举。
- 明确加载、空、错误和重试状态。

## 安全

- 除非经过净化且明确必要，否则不要使用 `dangerouslySetInnerHTML`。
- 对用户提供的富文本进行转义或净化。
- 不要记录 token、密码、身份证号、手机号、患者敏感数据或原始敏感响应。
- Token 存储和刷新行为应遵循现有请求/鉴权基础设施。
- 前端权限控制是用户体验控制；敏感授权必须由后端强制执行。

## 权限体验

- 保持路由守卫和角色元数据。
- 按现有权限模式隐藏或禁用未授权的菜单/按钮操作。
- 对破坏性或敏感操作要求确认。
- 业务规则要求时，对手机号或身份证号等敏感展示数据做脱敏。
