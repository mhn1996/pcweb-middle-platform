# Figma 转代码

用于将 Figma、生成式 UI、复制来的设计代码或导入的 mockup 转换为可维护的 PC Web 代码。

## 红线

- 不要在 `src/imports/**` 下新增实现。
- 不要路由到原始 Figma 导出的组件。
- 不要保留生成式噪音，例如 `data-name`、无意义的生成类名、未使用的 SVG path 模块、生成的字体声明，或冗长的绝对定位像素链。
- 不要用自定义页面级 UI 框架替换企业级 AntD 交互模型。
- 替换后不要保留旧页面实现和旧 imports。

## 工作流

1. 确认目标业务路由和正式页面目录。
2. 从设计中提取真实信息架构：
   - 页面标题和上下文
   - 筛选/搜索控件
   - 主要操作
   - 数据展示
   - 详情/编辑流程
   - 加载/空/错误状态
3. 使用 AntD 组件和现有项目布局重建页面。
4. 只有在能提升可读性时，才把局部区块移动到页面私有组件。
5. 尽可能用项目 token/config 替换原始设计颜色、间距和圆角。
6. 通过 `src/router/routeConfig.tsx` 接入路由。
7. 移除废弃实现、过期 imports 和未引用的设计产物。
8. 条件允许时运行 lint/build。

## 转换指南

- 表格通常应转换为 AntD `Table`，包含类型化 columns、分页、loading、row keys 和操作列。
- 搜索/筛选区通常应转换为 AntD `Form`，配合 `Space` 或响应式网格布局。
- 编辑/创建流程通常使用 `Modal` 或 `Drawer` 搭配 `Form`。
- 状态展示使用 `Tag`、`Badge`、`Typography.Text` 或语义化颜色 token。
- 指标展示优先使用 AntD 布局原语；只有 AntD 本身过于冗长时，才使用可复用的局部卡片。
- 图标应来自项目既有图标选择，除非设计要求特定资源。

## 验收标准

- 路由和鉴权行为保持稳定。
- 业务 UI 是 AntD First。
- 没有新增 `src/imports/**` 实现。
- 路由路径中没有原始生成式设计代码。
- 旧实现已移除或被完整替换。
- 涉及数据时，页面具备有意义的加载、空和失败状态。
